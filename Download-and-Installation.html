Pre-compiled executable program

Pre-compiled program does not require any external library or GROMACS and APBS package. These programs are standalone and without any dependency. Download, extract and use it.

Presently, these programs are only available for Linux x86_64 (64 bit).

Installation

Extract the package. Copy g_mmpbsa and energy2bfac from bin directory to /usr/local/bin or $HOME/bin. 

tar -zxvf g_mmpbsa.tar.gz
cd bin
sudo cp g_mmpbsa /usr/local/bin/.
sudo cp energy2bfac /usr/local/bin/.
or 
tar -zxvf g_mmpbsa.tar.gz
cd bin
cp g_mmpbsa $HOME/bin/.
cp energy2bfac $HOME/bin/.
or add g_mmpbsa/bin to PATH environment variable by adding following line in .bashrc file: 
export PATH=${PATH}:/path/to/g_mmpbsa/bin

Parallel support

g_mmpbsa inherits OpenMP parallel execution implemented in APBS 1.2.x or 1.3.x. This parallel support is enabled by default and all processors/core will be used during run-time.

To control the usage of processors/cores, number of threads can be changed by defining environment variable. For example in bash, one can write following command: 

 export OMP_NUM_THREADS=X
where X is number of core/processors.


Installation from Source-Code

Requirement

1. GROMACS 4.5.x/4.6.x
 Required Libraries : libgmx.a, libgmxana.a and libmd.a
or
 Required Libraries : libgmx.so, libgmxana.so and libmd.so

GROMACS should be compiled and installed from the source-code.

g_mmpbsa cannot use GPU and cannot be compiled with CUDA. If CUDA is in PATH variable, GROMACS-4.6.x compiled by default with CUDA for GPU acceleration. To avoid error in g_mmpbsa compilation, use -DGMX_GPU=off with cmake during installation of GROMACS-4.6.x.

Currently, double precision GROMACS is not supported.

2. APBS 1.2.x or 1.3.x
Required Libraries : libapbsmainroutines.a, libapbs.a, libapbsblas.a, 
                     libmaloc.a, libapbsgen.a and libz.a

APBS should be compiled and installed from the source-code. Downloaded binaries and libraries would not work as some required libraries are missing.

Note: During installation and execution of program, errors may appear. Solution of the errors are given below in list which will be regularly updated.

Installation

Please post problems in g_mmpbsa forum, we will try our best to provide the solution.

1. Installation of GROMACS

GROMACS-4.5.x
tar -zxvf gromacs-4.5.5.tar.gz
cd gromacs-4.5.5
./configure --prefix=/opt/gromacs
make
make install
GROMACS-4.5.7 and GROMACS-4.6.x
tar -zxvf gromacs-4.6.5.tar.gz
cd gromacs-4.6.5
mkdir build
cd build
cmake .. -DCMAKE_INSTALL_PREFIX=/opt/gromacs -DGMX_GPU=off
make
make install

2. Installation of APBS

tar -zxvf apbs-1.3-source.tar.gz
cd apbs-1.3-source
./configure --prefix=/opt/apbs
make
make install

3. Configure for g_mmpbsa

To get detailed options

./configure --help

Defining Path to GROMACS and APBS headers and libraries files

Path to GROMACS include directory

 --with-gmx-include=/opt/gromacs/include

Path to GROMACS Libraries files

 --with-gmx-lib=/opt/gromacs/lib

Path to APBS Libraries files

 --with-apbs-lib=/opt/apbs/lib

Path to APBS source directory

 --with-apbs=/path/to/apbs-1.3-source
For GROMACS-4.5.x
 ./configure --disable-gmx46 --with-gmx-include=/opt/gromacs/include --with-gmx-lib=/opt/gromacs/lib --with-apbs-lib=/opt/apbs/lib --with-apbs=/path/to/apbs-1.3-source
For GROMACS-4.6.x
 ./configure --enable-gmx46 --with-gmx-include=/opt/gromacs/include --with-gmx-lib=/opt/gromacs/lib --with-apbs-lib=/opt/apbs/lib --with-apbs=/path/to/apbs-1.3-source

During configure, libfftw3f.a will be searched in the  standard locations. If this file is not present in these locations, use environment variable LDFLAGS to define the path. For example in bash: 

 export LDFLAGS=-L/path/to/fftw3/lib

4. Compiling g_mmpbsa

 make

5. Installing g_mmpbsa

 make install

List of Errors

  • Static gfortran library missing 
     /usr/bin/ld: cannot find -lgfortran
    Please, install static gfortran library file " libgfortran.a ".
  • During execution of g_mmpbsa, error of shared object file. 
    g_mmpbsa: error while loading shared libraries: libgmx.so.8: cannot open shared object file: No such file or directory

    Define the path of libgmx.so, libmd.so and libgmxana.so files by LD_LIBRARY_PATH environment variable in terminal or .bashrc as following:

     export LD_LIBRARY_PATH=/opt/gromacs/lib
